# HPM community registry

**Modern Dashboard** is listed in the [Hubitat Package Manager community registry](https://github.com/HubitatCommunity/hubitat-packagerepositories). Users can install it from HPM search without adding a custom repository URL.

## Install (users)

1. Open **Hubitat Package Manager** on your hub.
2. **Install** (or search) → **Modern Dashboard** → install.
3. **Apps → Add User App → Modern Dashboard** → select devices → **Done**.

HPM enables OAuth and deploys all 12 File Manager assets automatically.

## Maintainer notes

- Keep `hubitat/repository.json` and `hubitat/packageManifest.json` on `master` — the registry points at `repository.json`.
- Package version and release notes live in `hubitat/packageManifest.json` (generated by `node build.mjs`).
- Do not change package or app UUIDs in the manifest after users have installed.
- After publishing a new version, users receive it via HPM **Update**. If assets look stale after **Match Up**, advise **Repair** on the package.

## Registry entry

The listing in [hubitat-packagerepositories](https://github.com/HubitatCommunity/hubitat-packagerepositories) references:

```json
{
  "name": "evdev Modern Dashboard",
  "location": "https://raw.githubusercontent.com/evdev/hubitat-modern-dashboard/master/hubitat/repository.json"
}
```

Users who installed via this custom URL before the registry merge can keep using HPM **Update** as usual, or remove the old custom-repository entry from HPM Settings — it is optional once the default repository is enabled.
